# Movie Filtering API

Сервис для загрузки, хранения и фильтрации данных о фильмах и сериалах на основе датасета Netflix.  
Реализована полная контейнеризация (Docker), авторизация по JWT, нормализация категорий и рейтингов, пагинация, фильтрация по множеству параметров.

## Обработка данных (ETL)

Исходный CSV содержит поля: `show_id`, `type`, `title`, `director`, `cast`, `country`, `date_added`, `release_year`, `rating`, `duration`, `listed_in`, `description`.

### Нормализация и очистка:
- Пропуски в `director`, `cast`, `country` заменены на пустую строку.
- Даты приведены к типу `date`.
- Категории (`listed_in`, например `"Children & Family Movies, Comedies"`) выделены в отдельную таблицу `categories`, связь с фильмами через `movie_category` (many to many).
- Рейтинги вынесены в таблицу `ratings` (связь по foreign key).
- Поле `show_id` хранится как строка, уникальное ограничение.
- Перед вставкой проверяется нет ли уже записи с таким `show_id`.

## API эндпоинты

### Регистрация и авторизация

| Метод | Эндпоинт       | Описание                                      | Тело запроса                              |
|-------|----------------|-----------------------------------------------|-------------------------------------------|
| POST  | `/register`    | Регистрация нового пользователя + выдача JWT  | `{"username": "user", "password": "pass"}` |
| POST  | `/login`       | Логин (OAuth2) и получение JWT                | `form-data` с `username` и `password`     |

### Фильтрация и поиск фильмов 

`GET /movies` – список фильмов/сериалов с пагинацией и фильтрацией.

**Параметры запроса (query string)**:

| Параметр        | Тип      | Описание                                                     |
|-----------------|----------|--------------------------------------------------------------|
| `search`        | string   | Частичное совпадение с `title`            |
| `type`          | string   | `Movie` или `TV Show`                                        |
| `category`      | string   | Название категории (например, `Comedies`, `Kids' TV`)         |
| `rating`        | string   | Рейтинг (например, `TV-PG`, `TV-MA`, `R`)                    |
| `release_year`  | integer  | Год выпуска                                                  |
| `limit`         | integer  | Количество результатов на странице (default 20)             |
| `offset`        | integer  | Смещение для пагинации                                       |

**Пример запроса с фильтрами:**  
`GET /movies?search=King&type=Movie&category=Comedies&rating=TV-PG&limit=10`

**Ответ (JSON):**
```json
{
  "total": 42,
  "items": [
    {
      "id": 1,
      "show_id": "81145628",
      "title": "Norm of the North: King Sized Adventure",
      "type": "Movie",
      "release_year": 2019,
      "rating": "TV-PG",
      "categories": [
        {"id": 5, "name": "Children & Family Movies"},
        {"id": 12, "name": "Comedies"}
      ],
      ...
    }
  ]
}
```

## Примеры запросов (curl)

### Регистрация нового пользователя
```bash
curl -X POST http://localhost:8000/register \
  -H "Content-Type: application/json" \
  -d '{"username": "alice", "password": "secret"}'
```
**Ответ:** `{"access_token": "eyJ...", "token_type": "bearer"}`

### Логин (OAuth2)
```bash
curl -X POST http://localhost:8000/login \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=alice&password=secret"
```

### Получение фильмов (с авторизацией)
```bash
TOKEN="eyJ..."   # скопировать из ответа логина

curl -X GET "http://localhost:8000/movies?search=The&type=Movie&limit=5" \
  -H "Authorization: Bearer $TOKEN"
```

## Особенности 

- **Безопасное хэширование паролей:** используется `passlib` с `pbkdf2_sha256`
- **JWT токен** содержит username, время истечения. Все защищённые эндпоинты проверяют токен через зависимость `get_current_user`.
- **Нормализованная схема БД:** категории вынесены отдельно, что позволяет фильтровать по категориям и избегает дублирования строк.
- **Обработка дубликатов при повторном запуске ETL:** скрипт проверяет существование `show_id` перед вставкой что позволяет перезапускать контейнер без ошибок `UniqueViolation`.
- **Docker-конфигурация:** сервис `web` ждёт 5 секунд старта БД, затем выполняет создание таблиц и ETL, после чего запускает Uvicorn. 
- **Простой веб-интерфейс:** в `static/index.html` включает авторизацию, форма поиска, отображение результатов. Демонстрирует работу API через fetch с токенами.

## Тестирование

1. **Проверка API через Swagger:**  
   Открыть `http://localhost:8000/docs`, авторизоваться через `/login` (кнопка `Authorize`), затем выполнить `/movies` с любыми параметрами.

2. **Веб-интерфейс:**  
   Перейти на `http://localhost:8000/static/index.html`, ввести логин и пароль, нажми `Login/Register`. Заполнить поля и нажать `Search`. Результаты загрузятся без перезагрузки страницы.

3. **Проверка ограничений:**  
   - зарегистрировать уже существующего пользователя - `400` с ошибкой `Username already exists`.
   - запросить `/movies` без токена -  `401 Unauthorized`.
   - проверить фильтр по несуществующей категории - вернётся `total: 0`.
   - проверить что пагинация работает (параметры `limit`, `offset`).
